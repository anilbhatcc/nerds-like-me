What is an AI Skill?

A recipe tells a chef how to cook.

A skill tells an AI how to work.

Claire's key insight: "Multiple specialized agents beat one general-purpose agent." Skills are how you specialize.

Anthropic's framing: "Don't build agents, build skills instead." Skills are the modular building blocks — agents are just the runtime.

👨‍🍳 → 📋 → 🤖

Chef + Recipe = Consistent Dish
Agent + Skill = Focused Output

⚠️ Non-deterministic: Same skill may produce slightly different output each run. That's normal — iterate.

Nerds Like Me 🦞  |  Pop-up Hackathon
The Chef & Recipe model is the mental model for tonight. Claire Vo's insight is crucial — she didn't build one mega-agent. She built 9 specialized ones, each with a focused skill set. Skills are the modular building blocks. Without skills, the chef guesses. With skills, it executes. Caveat: unlike real recipes, AI is non-deterministic. How We Just Built That

The Record-Transcribe-Refine method:

1. Record — How do you do this task right now? (screen recording, voice memo, or written steps)
2. Transcribe — Turn that into plain language instructions
3. Refine — Test it. Fix gaps. Iterate until it matches what you'd do

💡 You're not inventing from scratch — you're documenting what you already know.

Nerds Like Me 🦞  |  Pop-up Hackathon
This is the beginner onboarding path. Don't start from a blank page — start from something you already do. For the hackathon: think of something annoying you do manually. That's your first skill. Record-Transcribe-Refine works even if you just describe the steps out loud and have AI transcribe them. Claire's first "skill" was probably just documenting how she manages her family calendar — and now that agent runs autonomously.

Anatomy of a Skill

A working skill can be 10 lines of markdown.

---
name: my-skill
description: What this skill does and when to
  trigger it. This is how the agent finds it.
---

# My Skill

Steps to execute when this skill is active:

1. Do X
2. Then do Y
3. Return result in format Z

📁 Skills live in: ~/.openclaw/workspace/skills/<name>/SKILL.md

Nerds Like Me 🦞  |  Pop-up Hackathon
Two parts: YAML frontmatter (metadata — name and description) and markdown body (the actual instructions). The description is critical — it's how the agent discovers and decides to use your skill. The markdown body is the recipe. Keep it focused, step-by-step. Optionally add scripts/, references/, and assets/ folders for more advanced skills. Anthropic's Complete Guide to Building Skills (PDF) walks through this anatomy in detail — it's in the resources.

Common Gotchas 🚧

01Skills should DO things, not TEACH things. Give execution steps, not a Wikipedia article.
02Keep scope narrow. One skill, one job. Think Claire: 9 agents, not 1.
03Test early with real inputs. Don't wait for perfection.
04The description field is how the agent discovers your skill. Make it specific.
05First output won't be perfect — iterate on the instructions, not the prompt.
Nerds Like Me 🦞  |  Pop-up Hackathon
Walk through each one briefly. The biggest mistake is #1 — people write a textbook instead of a recipe. Keep it actionable. #4 is the most common reason skills don't trigger — vague descriptions. Note the Claire reference on #2: she didn't build one agent that does everything. She built 9 specialized agents, each doing one thing well. That's the model. Anthropic's DeepLearning.AI course covers these gotchas in depth — see resources.

When Things Go Wrong 🔧

Skill not triggeringDescription doesn't match your request → Make the description more specific
Output format is wrongInstructions too vague → Add explicit format examples and constraints
Agent ignores the skillDescription doesn't cover this use case → Rewrite to match the words you'd naturally say
Output is way offScope too broad → Narrow the skill to one specific job

💡 90% of problems come down to: description matching or instruction clarity.

Nerds Like Me 🦞  |  Pop-up Hackathon
Quick troubleshooting guide. Most issues boil down to two things: the agent can't find your skill (description problem) or it found it but the instructions aren't clear enough. Fix those two things first before anything else.

What Skills Can't Do

❌ Guarantee consistent output (non-deterministic)

❌ Execute on their own without an agent runtime

❌ Replace human judgment

❌ Run without you checking the output

Claire's advice: "Don't install on your main computer first." Start with non-critical tasks. Handle with care.

Force-multipliers, not autonomous workers.

Nerds Like Me 🦞  |  Pop-up Hackathon
Be honest about limitations. Skills are tools, not replacements. Claire's advice is perfect here — she literally deleted her family calendar on her first try because she didn't start small. Now she runs 9 agents and handles them with care. Always start with non-critical tasks, review output, and apply judgment. The non-determinism is a feature, not a bug — but it means you always need a human in the loop. Security matters — these are powerful tools, treat them that way.
